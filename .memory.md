## Key discoveries

### 1. Los scrapers ofuscados son para Hermes, NO para Node.js
- Los 61 archivos Hermes en `providers/` estan ofuscados con `__async`, `__defProp`, etc. (mĂˇs 1 `alfa-providers.js` = 62 archivos total).
- Diseniados para **Hermes** (motor JS de React Native en Nuvio).
- **NO se pueden ejecutar en el backend.** Son para ejecucion local en el dispositivo.

### 2. Nuvio 0.7.5 usa protocolo Stremio
- Solo existe "Addons" (Stremio protocol), no "Local Scrapers".
- Campos obligatorios: `id`, `version`, `name`, `logo`, `resources`, `types`, `idPrefixes`.
- Formato de resources: array de objetos `[{name, types, idPrefixes}]`.

### 3. El campo "scrapers" en el manifest
- Si se incluye, Nuvio los ejecuta **localmente en el dispositivo**.
- Se puede omitir via config (`enableLocal: false`).
- Se envia condicionalmente en el manifest segun `?configured=`.

### 5. Pigamer37 como proxy de anime
- El addon original de Pigamer37 usa `|` (pipe) en IDs de catalogo: `animeflv|onair`, `animeflv|search`
- Content IDs usan `:` (colon): `animeflv:slug`, `animeav1:slug`
- Soporta: AnimeFLV, AnimeAV1, TioAnime, Henaojara
- IDs: `animeflv:`, `animeav1:`, `henaojara:`, `tioanime:`, `anilist:`, `kitsu:`, `mal:`, `anidb:`
- **IMPORTANTE:** Pigamer37 espera IDs con `%7C` (pipe encodeado), NO `%3A` (colon encodeado). `encodeURIComponent` encodea `:` como `%3A` â€” hay que usar `fixPigamerId()` que reemplaza `:` por `%7C`.

### 6. Stremio stream format (Pigamer37 reference)
- `name`: `"SourceName\nServerName"` (multilinea con `\n`)
- `title`: `"Episode Title\nâš™ď¸Ź ServerName\nđź”— url\nđź‡Żđź‡µđź‡Şđź‡¸"` (info detallada)
- `behaviorHints.notWebReady: true` para streams internos
- `behaviorHints.bingeGroup`: `"source|server"` para autoplay
- `externalUrl` (sin `url`) para streams que abren en navegador
- `proxyHeaders` para headers custom al fetchear el video

### 7. Stremio catalog format (Pigamer37 reference)
- Catalogs con `extra: [{ name: "genre", options: [...], optionsLimit: 1 }]` para generos
- `extra: [{ name: "search", isRequired: true }]` para busqueda
- `skip` para paginacion: `page = Math.floor(skip / pageSize) + 1`
- Genre catalog sin search: `id: "animeflv|genres"`, `extra.genre.isRequired: true`
- Los `idPrefixes` deben incluir todos los prefixes usados en items (incluyendo `tmdb-genre:`)

### 8. DonTorrent: Doble PoW (Anubis + API interno) + .torrent download

DonTorrent tiene un sistema de protecciĂłn de 2 capas:

**Capa 1 - Anubis (Cloudflare):** PoW SHA256 con dificultad 5 (~50K-1M iteraciones). Se resuelve en `bypassAnubisChallenge()` en engine.js. Cookie `browser-pow-auth` cacheada por dominio.

**Capa 2 - PoW interno (`/api_validate_pow.php`):**
- `POST { action: "generate", content_id, tabla }` â†’ recibe `challenge` (SHA256 hex)
- Resolver: SHA256(challenge + nonce) con 3 ceros hex leading (~3K iteraciones, ~7ms)
- `POST { action: "validate", challenge, nonce }` â†’ recibe `download_url` (protocol-relative URL al .torrent)
- Descargar .torrent â†’ parsear bencode â†’ extraer infoHash (SHA1 del bloque `info`)

**Click handler JS (`don_download.js`):**
```javascript
// 1. POST /api_validate_pow.php { action: 'generate', content_id, tabla }
// 2. computeProofOfWork(challenge, difficulty=3) â€” SHA256(challenge+nonce), leading zeros
// 3. POST /api_validate_pow.php { action: 'validate', challenge, nonce }
// 4. window.location.href = result.download_url
```

**URL estructural:**
- PelĂ­culas: `/pelicula/{content_id}/{slug}`
- Series (temporada): `/serie/{show_id}/{season_id}/{slug}`
- Tabla de episodios: `<table.table>` con filas `<tr>` â†’ `1x01 - Titulo`, botĂłn `.protected-download` con `data-content-id` y `data-tabla="series"`, clave "Sin ContraseĂ±a"
- BĂşsqueda: solo POST a `/buscar` con `valor={query}`, NO GET

**Streams resultantes:** .torrent file â†’ infoHash extraĂ­do â†’ stream con `{ url: dl_url, infoHash, filename, sources: ['dht:'+infoHash] }`

## Config protocol

El panel `/configure` genera una URL con los ajustes codificados en base64:

```
https://ovnivers.onrender.com/manifest.json?configured=<json_base64>
```

El backend decodifica `configured` y:
1. Filtra `catalogs`, `resources`, `types`, `idPrefixes` en el manifest
2. Omite `scrapers` si `enableLocal: false`
3. Filtra streams por `quality` en `/stream`
4. Bloquea requests de tipos deshabilitados

Campos configurables:
```json
{
  "enableMovies": true,
  "enableSeries": true,
  "enableAnime": true,
  "quality": "all",
  "langs": ["en", "es", "ja", ...],
  "enableBackend": true,
  "enableLocal": true
}
```
