# Ovnivers — Project Memory

## What is this
Addon para **Nuvio 0.7.5** que combina 61 scrapers de streaming de [All-in-One-Nuvio](https://github.com/D3adlyRocket/All-in-One-Nuvio) adaptados al protocolo Stremio con backend en **Render.com**. Proxy de anime integrado via Pigamer37 (AnimeFLV/AnimeAV1/TioAnime/Henaojara).

---

## Architecture (v1.6.1)

```
Nuvio App (0.7.5)
  |-- Settings > Addons
  |     URL: https://ovnivers.onrender.com/manifest.json
  |     URL configurada: .../?configured=<base64>
  |
  |-- Backend: Render.com (Node.js / Express)
  |     |-- /manifest.json           → Stremio manifest (respeta config)
  |     |-- /stream/:type/:id.json   → Backend scrapers (6) + Pigamer37 proxy
  |     |-- /catalog/:type/:id.json  → 18 catalogs (TMDb + anime)
  |     |-- /meta/:type/:id.json     → TMDb metadata + anime metadata
  |     |-- /configure               → Panel moderno con generate URL
  |     |-- /                        → Health check
  |
  |-- El campo "scrapers" en manifest → Nuvio los descarga y ejecuta LOCALMENTE
        en el dispositivo (motor Hermes). NO en el servidor.
        Incluye 61 scrapers originales + 1 scraper unificado "alfa-providers"
        con 80+ providers del addon Alfa de Kodi.
```

## Key discoveries

### 1. Los scrapers ofuscados son para Hermes, NO para Node.js
- Los 61 archivos `.js` en `providers/` estan ofuscados con `__async`, `__defProp`, etc.
- Diseniados para **Hermes** (motor JS de React Native en Nuvio).
- **NO se pueden ejecutar en el backend.** Son para ejecucion local en el dispositivo.

### 2. Nuvio 0.7.5 usa protocolo Stremio
- Solo existe "Addons" (Stremio protocol), no "Local Scrapers".
- Campos obligatorios: `id`, `version`, `name`, `logo`, `resources`, `types`, `idPrefixes`.
- Formato de resources: array de objetos `[{name, types, idPrefixes}]`.

### 3. El campo "scrapers" en el manifest
- Si se incluye, Nuvio los ejecuta **localmente en el dispositivo**.
- Se puede omitir via config (`enableLocal: false`).
- Se envia condicionalmente en el manifest segun `?configured=`.

### 5. Pigamer37 como proxy de anime
- El addon original de Pigamer37 usa `|` (pipe) en IDs de catalogo: `animeflv|onair`, `animeflv|search`
- Content IDs usan `:` (colon): `animeflv:slug`, `animeav1:slug`
- Soporta: AnimeFLV, AnimeAV1, TioAnime, Henaojara
- IDs: `animeflv:`, `animeav1:`, `henaojara:`, `tioanime:`, `anilist:`, `kitsu:`, `mal:`, `anidb:`
- **IMPORTANTE:** Pigamer37 espera IDs con `%7C` (pipe encodeado), NO `%3A` (colon encodeado). `encodeURIComponent` encodea `:` como `%3A` — hay que usar `fixPigamerId()` que reemplaza `:` por `%7C`.

### 6. Stremio stream format (Pigamer37 reference)
- `name`: `"SourceName\nServerName"` (multilinea con `\n`)
- `title`: `"Episode Title\n⚙️ ServerName\n🔗 url\n🇯🇵🇪🇸"` (info detallada)
- `behaviorHints.notWebReady: true` para streams internos
- `behaviorHints.bingeGroup`: `"source|server"` para autoplay
- `externalUrl` (sin `url`) para streams que abren en navegador
- `proxyHeaders` para headers custom al fetchear el video

### 7. Stremio catalog format (Pigamer37 reference)
- Catalogs con `extra: [{ name: "genre", options: [...], optionsLimit: 1 }]` para generos
- `extra: [{ name: "search", isRequired: true }]` para busqueda
- `skip` para paginacion: `page = Math.floor(skip / pageSize) + 1`
- Genre catalog sin search: `id: "animeflv|genres"`, `extra.genre.isRequired: true`
- Los `idPrefixes` deben incluir todos los prefixes usados en items (incluyendo `tmdb-genre:`)

## Config protocol

El panel `/configure` genera una URL con los ajustes codificados en base64:

```
https://ovnivers.onrender.com/manifest.json?configured=<json_base64>
```

El backend decodifica `configured` y:
1. Filtra `catalogs`, `resources`, `types`, `idPrefixes` en el manifest
2. Omite `scrapers` si `enableLocal: false`
3. Filtra streams por `quality` en `/stream`
4. Bloquea requests de tipos deshabilitados

Campos configurables:
```json
{
  "enableMovies": true,
  "enableSeries": true,
  "enableAnime": true,
  "quality": "all",
  "langs": ["en", "es", "ja", ...],
  "enableBackend": true,
  "enableLocal": true
}
```

### ✅ FIXED — Backend EZTV y Cuevana2 removidos
- **Fix aplicado:** no funcionan desde Render (403/fetch failed), quitados del array activo

### ✅ FIXED — 2embed devuelve HTML, no JSON → scrapers no extraen streams (2026-06-12)
- **Fix aplicado:** `parse2embedHTML()` extrae m3u8/mp4/iframe/data-src/json-block URLs via regex
- `scrape2embedVesy` y `scrape2embedVsrc` ahora detectan `typeof data === 'string'` y usan el parser HTML
- 6 patrones regex: m3u8, mp4, iframe src, JS assignments, data-src attributes, JSON blocks

### ✅ FIXED — Stremio manda extra params en URL path, no en query string (2026-06-12)
- **Fix aplicado:** rutas catch-all `/:type/:id/:extra(*).json` para stream, catalog y meta
- `parsePathExtras()` convierte `genre=28&skip=20` del path en `req.query`
- Handlers refactorizados a funciones `handleStream()`, `handleCatalog()`, `handleMeta()`

### ✅ FIXED — Anime catalogs usan type incorrecto (2026-06-12)
- **Fix aplicado:** todos los catalogs de anime cambiados a `type: "series"` (manifest.json + server.js)
- Coincide con Pigamer37 y Stremio espera `series` para contenido episodico

### ✅ FIXED — Genre catalogs no tienen `options` en manifest extra (2026-06-12)
- **Fix aplicado:** `loadGenres()` carga generos TMDB (movie + tv) con cache 24h
- Manifest genera `extra: [{ name: "genre", options: [...], optionsLimit: 1 }]` dinamicamente
- Fallback estatico en manifest.json con lista completa de generos

### ✅ FIXED — El addon no aparece como provider en catalogs de otros addons (2026-06-12)
- **Fix aplicado:** 2embed scrapers ahora devuelven streams reales (HTML parser)
- Manifest ya tenia `tt` en stream idPrefixes — el problema era que los scrapers no devolvian nada

### ✅ FIXED — Catalogo "By Year" no funciona sin search param (2026-06-12)
- **Fix aplicado:** `isRequired: true` en manifest + fallback a `new Date().getFullYear()` en server

### ✅ FIXED — `fixPigamerType()` y `cheerio` fallback son codigo muerto (2026-06-12)
- **Fix aplicado:** `fixPigamerType()` eliminado. `scrapeEZTV`, `scrapeCuevana2Espanol`, `getTitle` eliminados
- `scrapePoseidonHD` refactorizado para usar `getTMDbMeta()` directamente

### ✅ FIXED — Pigamer37 double-encoding bug (2026-06-12)
- **Fix aplicado:** `fixPigamerId()` ahora retorna `|` literal (no `%7C`), `encodeURIComponent` lo convierte a `%7C`
- Antes retornaba `%7C` literal → `encodeURIComponent` encodeaba `%` → `%257C` (doble encoding)

### ✅ FIXED — Pigamer37 ID encoding: `:` vs `|` confusion (2026-06-12, v1.6.1)
- **BUG RAÍZ de todo el fallo de anime:** `fixPigamerId()` convertía `:` → `|` para TODOS los IDs
- **Realidad:** Pigamer37 usa `|` solo en catalog IDs (`animeflv|onair`), pero `:` en stream/meta IDs (`animeflv:one-piece`)
- `encodeURIComponent` encodea `:` como `%3A` y `|` como `%7C`
- Ovnivers enviaba `animeflv%7Cone-piece` → Pigamer37 devolvía `{"streams":[],"message":"Failed getting Anime info"}`
- **Fix:** `fixPigamerId()` ahora es no-op (`return id`). `encodeURIComponent` maneja la codificación correcta automáticamente
- Catalog IDs con `|` se encodean a `%7C` correctamente. Stream/meta IDs con `:` se encodean a `%3A` correctamente
- **Verificado:** `animeflv%3Aone-piece` → Pigamer37 devuelve 30+ streams reales

### ✅ FIXED — Anime catalog detection con IDs sin colon (2026-06-12)
- **Fix aplicado:** `isAnimeId(cleanId + ':')` en handleCatalog para IDs como `animeflv` (sin `:`)

### ✅ FIXED — Anime cross-reference IDs no se resuelven (2026-06-12)
- **Fix aplicado:** `resolveAnimeId()` consulta `/meta` de Pigamer37 para convertir `kitsu:`, `mal:`, `anilist:`, `anidb:` a IDs fuente (`animeflv:`, etc.)
- Stream handler ahora resuelve IDs antes de proxy a Pigamer37
- Esto permite que el addon aparezca como provider cuando otros addons (Anime Kitsu) devuelven IDs cross-reference

### ✅ FIXED — idPrefixes incorrectos por recurso (2026-06-12)
- **Fix aplicado:** `stream` ya no incluye `tmdb-genre:` (no tiene sentido para streams)
- `catalog` ya no incluye `tt` (no tiene sentido para catalogs)
- Cada recurso tiene sus idPrefixes especificos

### ✅ FIXED — Compatibilidad con Stremio (2026-06-12)
- **Fix aplicado:** `types` incluye `"other"` para compatibilidad con addons que usan ese tipo
- `behaviorHints.newEpisodeNotifications: true` para notificaciones de episodios
- `idPrefixes` por recurso limpios y correctos
