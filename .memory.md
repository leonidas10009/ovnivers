# Ovnivers — Project Memory

## What is this
Addon para **Stremio** que combina torrent indexers + Puppeteer anime resolvers + Pigamer37 + Alfa + Hermes. Backend en **Coolify** (self-hosted, 1.5GB RAM, Pentium + 6GB server).

> **Última sesión (2026-06-30 #10):** Migración completa de Render.com → Coolify. Dockerfile con Node 20 + Chromium + curl. Cloudflare Tunnel para exponer el addon (ngrok free solo permite 1 dominio). Memory watchdog corregido: compara contra heap limit (768MB) en vez de heapTotal. BrowserPool aumentado a 2 browsers. Heap: 768MB. Puerto 3000 expuesto al host.

## Key discoveries

### 1. Los scrapers ofuscados son para Hermes, NO para Node.js
- Los 61 archivos Hermes en `providers/` estan ofuscados con `__async`, `__defProp`, etc. (más 1 `alfa-providers.js` = 62 archivos total).
- Diseniados para **Hermes** (motor JS de React Native en Nuvio).
- **NO se pueden ejecutar en el backend.** Son para ejecucion local en el dispositivo.

### 2. Nuvio 0.7.5 usa protocolo Stremio
- Solo existe "Addons" (Stremio protocol), no "Local Scrapers".
- Campos obligatorios: `id`, `version`, `name`, `logo`, `resources`, `types`, `idPrefixes`.
- Formato de resources: array de objetos `[{name, types, idPrefixes}]`.

### 3. El campo "scrapers" en el manifest
- Si se incluye, Nuvio los ejecuta **localmente en el dispositivo**.
- Se puede omitir via config (`enableLocal: false`).
- Se envia condicionalmente en el manifest segun `?configured=`.

### 5. Pigamer37 como proxy de anime
- El addon original de Pigamer37 usa `|` (pipe) en IDs de catalogo: `animeflv|onair`, `animeflv|search`
- Content IDs usan `:` (colon): `animeflv:slug`, `animeav1:slug`
- Soporta: AnimeFLV, AnimeAV1, TioAnime, Henaojara
- IDs: `animeflv:`, `animeav1:`, `henaojara:`, `tioanime:`, `anilist:`, `kitsu:`, `mal:`, `anidb:`
- **IMPORTANTE:** Pigamer37 espera IDs con `%7C` (pipe encodeado), NO `%3A` (colon encodeado). `encodeURIComponent` encodea `:` como `%3A` — hay que usar `fixPigamerId()` que reemplaza `:` por `%7C`.

### 6. Stremio stream format (Pigamer37 reference)
- `name`: `"SourceName\nServerName"` (multilinea con `\n`)
- `title`: `"Episode Title\n⚙️ ServerName\n🔗 url\n🇯🇵🇪🇸"` (info detallada)
- `behaviorHints.notWebReady: true` para streams internos
- `behaviorHints.bingeGroup`: `"source|server"` para autoplay
- `externalUrl` (sin `url`) para streams que abren en navegador
- `proxyHeaders` para headers custom al fetchear el video

### 7. Stremio catalog format (Pigamer37 reference)
- Catalogs con `extra: [{ name: "genre", options: [...], optionsLimit: 1 }]` para generos
- `extra: [{ name: "search", isRequired: true }]` para busqueda
- `skip` para paginacion: `page = Math.floor(skip / pageSize) + 1`
- Genre catalog sin search: `id: "animeflv|genres"`, `extra.genre.isRequired: true`
- Los `idPrefixes` deben incluir todos los prefixes usados en items (incluyendo `tmdb-genre:`)

### 8. DonTorrent: Doble PoW (Anubis + API interno) + .torrent download

DonTorrent tiene un sistema de protección de 2 capas:

**Capa 1 - Anubis (Cloudflare):** PoW SHA256 con dificultad 5 (~50K-1M iteraciones). Se resuelve en `bypassAnubisChallenge()` en engine.js. Cookie `browser-pow-auth` cacheada por dominio.

**Capa 2 - PoW interno (`/api_validate_pow.php`):**
- `POST { action: "generate", content_id, tabla }` → recibe `challenge` (SHA256 hex)
- Resolver: SHA256(challenge + nonce) con 3 ceros hex leading (~3K iteraciones, ~7ms)
- `POST { action: "validate", challenge, nonce }` → recibe `download_url` (protocol-relative URL al .torrent)
- Descargar .torrent → parsear bencode → extraer infoHash (SHA1 del bloque `info`)

**Click handler JS (`don_download.js`):**
```javascript
// 1. POST /api_validate_pow.php { action: 'generate', content_id, tabla }
// 2. computeProofOfWork(challenge, difficulty=3) — SHA256(challenge+nonce), leading zeros
// 3. POST /api_validate_pow.php { action: 'validate', challenge, nonce }
// 4. window.location.href = result.download_url
```

**URL estructural:**
- Películas: `/pelicula/{content_id}/{slug}`
- Series (temporada): `/serie/{show_id}/{season_id}/{slug}`
- Tabla de episodios: `<table.table>` con filas `<tr>` → `1x01 - Titulo`, botón `.protected-download` con `data-content-id` y `data-tabla="series"`, clave "Sin Contraseña"
- Búsqueda: solo POST a `/buscar` con `valor={query}`, NO GET

**Streams resultantes:** .torrent file → infoHash extraído → stream con `{ url: dl_url, infoHash, filename, sources: ['dht:'+infoHash] }`

### ✅ FIXED — v1.6.5: Proxy no llegaba a los providers (CRÍTICO)

- **BUG RAÍZ:** `PROXY_URL` solo se usaba en `fetchAPI()` de server.js. Los 62 providers y Alfa usan `fetch()` global directamente → sus requests iban sin proxy → IP de Render bloqueada por todos los sitios → 0 streams.

- **Cloudflare Worker:** URL rewrite `?url=...` para cada request.
- **HTTP proxy:** `HttpsProxyAgent` inyectado en cada `fetch()`.
- **fetchAPI simplificado:** Ya no tiene lógica de proxy duplicada (global.fetch lo maneja).
- **Resultado:** 15/52 providers ahora completan exitosamente desde Render (antes 0). Health endpoint muestra stats por provider.

### ✅ FIXED — v1.6.5: Timeout global descartaba todos los resultados

- **BUG RAÍZ:** `Promise.race([Promise.allSettled(streamTasks), timeout])` — cuando el timeout ganaba, `rawStreams` quedaba vacío. El log decía "returning partial results" pero devolvía `[]`.
- **Fix:** Resultados se acumulan incrementalmente en `rawStreams` conforme cada task completa. El timeout solo corta la espera, no descarta lo ya recolectado.

## Config protocol

El panel `/configure` genera una URL con los ajustes codificados en base64:

```
https://<tunnel>/manifest.json?configured=<json_base64>
```

El backend decodifica `configured` y:
1. Filtra `catalogs`, `resources`, `types`, `idPrefixes` en el manifest
2. Omite `scrapers` si `enableLocal: false`
3. Filtra streams por `quality` en `/stream`
4. Bloquea requests de tipos deshabilitados

Campos configurables:
```json
{
  "enableMovies": true,
  "enableSeries": true,
  "enableAnime": true,
  "quality": "all",
  "langs": ["en", "es", "ja", ...],
  "enableBackend": true,
  "enableLocal": true
}
```

- Catalog IDs con `|` se encodean a `%7C` correctamente. Stream/meta IDs con `:` se encodean a `%3A` correctamente
- **Verificado:** `animeflv%3Aone-piece` → Pigamer37 devuelve 30+ streams reales

#### 1. TMDB_KEY inválida → 0 streams
- **Causa:** La variable `TMDB_KEY` en Coolify se configuró con un JWT en vez de la key real. Se borró la variable para que use el fallback hardcodeado (`d80ba92bc7cefe3359668d30d06f3305`), pero el deploy aún no se completó.
- **Solución:** Borrar `TMDB_KEY` de las env vars de Coolify (ya hecho). El código tiene fallback hardcodeado que sí funciona. Esperar a que el deploy termine.
- **Verificación:** `curl https://<tunnel>/stream/movie/tt0133093.json` debe devolver >0 streams.

#### 2. Nuvio no instala el addon
- **Manifest actual:** v1.14.8, JSON válido, CORS *, Content-Type correcto, 200 OK.
- **Posibles causas:**
  - Nuvio requiere `scrapers` en el manifest (solo se incluye con `exposeLocalScrapers=true`)
  - Nuvio espera `catalogs` no vacío (el addon es stream-only, `catalogs: []`)
  - La URL de Cloudflare Tunnel (`trycloudflare.com`) no es aceptada por Nuvio
  - Nuvio necesita HTTPS con certificado válido (Cloudflare Tunnel lo tiene)
